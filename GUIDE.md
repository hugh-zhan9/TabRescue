## 快速开始

### 第一次使用

1. **安装扩展** - 参考 [INSTALL.md](./INSTALL.md)
2. **打开 popup** - 点击浏览器工具栏中的归屿图标
3. **创建第一个快照** - 点击"📸 立即保存"

### 恢复会话

1. 在快照列表中找到要恢复的快照
2. 点击"恢复"按钮
3. 确认恢复（如果需要）

## 功能详解

### 1. 实时保存

归屿会自动监听您的浏览器活动：

- **标签页变化**: 打开、关闭、更新、移动
- **窗口变化**: 创建、关闭、聚焦变化
- **自动保存**: 每 5 分钟自动生成快照（可配置）

### 2. 快照列表

快照按时间倒序排列，每个快照显示：

- **相对时间**: "刚刚"、"5 分钟前"、"2 小时前"
- **窗口数量**: 快照包含的窗口数
- **标签数量**: 快照包含的标签页数
- **预览**: 前几个标签页的标题

### 3. URL 去重策略

在设置中可以配置三种策略：

#### strict（严格去重）
- 相同 URL 全局只保存一份
- 适合：标签页重复度高的用户

#### per-window（按窗口去重，推荐）
- 同一窗口内去重，跨窗口可重复
- 适合：大多数用户

#### none（完全不去重）
- 每个标签页独立记录
- 适合：需要 100% 忠实恢复的用户

### 4. 存储模式

#### Level 1: 纯扩展（默认）
- 使用 chrome.storage.local
- 容量：约 5-10MB
- 无需额外配置

#### Level 2: 本地 SQLite
- 需要安装 Native Host
- 容量：磁盘限制
- 适合重度用户

#### Level 3: 远程数据库
- 需要 PostgreSQL 或 MySQL
- 支持跨设备同步
- 适合多设备用户

## 常见问题

### Q: 为什么有些标签页没有被保存？

以下类型的标签页不会被保存：

- 隐私模式（Incognito）窗口
- 浏览器内部页面（chrome://, about:, edge://）
- 扩展页面（chrome-extension://）
- 新标签页（about:newtab）
- 特殊协议（data:, javascript:, ftp://）

### Q: 恢复时能否保留标签页的滚动位置？

不能。归屿只保存 URL 和标题，不保存页面内部状态（滚动位置、表单内容、登录态等）。

### Q: 如何备份我的快照数据？

**Level 1 模式:**
- 数据存储在 chrome.storage.local
- 可通过 Chrome 同步功能跨设备同步

**Level 2 模式:**
- 数据存储在 `~/.tabrescue/data.db`
- 可直接复制该文件进行备份

**Level 3 模式:**
- 数据在远程数据库
- 通过数据库备份机制备份

### Q: 恢复失败怎么办？

1. 检查网络连接
2. 检查 URL 是否仍然有效
3. 某些网站可能阻止在后台打开

### Q: 可以在多个浏览器上使用吗？

可以。每个浏览器的数据是隔离的：

- Chrome 扩展只能访问 Chrome 的数据
- Edge 扩展只能访问 Edge 的数据
- Firefox 扩展只能访问 Firefox 的数据

## 最佳实践

### 1. 定期手动保存

在以下场景建议手动保存：

- 完成一个重要工作阶段
- 准备关闭大量标签页
- 切换工作上下文

### 2. 合理配置快照数量

- 轻度用户：10-20 个
- 重度用户：30-50 个
- 注意存储容量限制

### 3. 使用标签页分组

配合浏览器的标签页分组功能：

1. 将相关标签页分组
2. 为每个工作组创建快照
3. 恢复时更有针对性

### 4. 定期清理

- 删除不需要的旧快照
- 清理已删除的标签页数据
- 保持存储空间充足

### 5. 跨设备同步（Level 3）

配置远程数据库后：

1. 在所有设备安装归屿
2. 配置相同的数据库连接
3. 即可实现跨设备同步
