Read the engineering guidelines before starting non-UI development work.

Execute these steps:
1. Read `.trellis/spec/engineering/index.md` to understand available guidelines
2. Based on your task, read the relevant guideline files:
   - Error handling → `.trellis/spec/engineering/error-handling.md`
   - Logging → `.trellis/spec/engineering/logging-guidelines.md`
   - Storage changes → `.trellis/spec/engineering/storage-guidelines.md`
   - Browser compatibility work → `.trellis/spec/engineering/compatibility-guidelines.md`
3. Understand the coding standards and patterns you need to follow
4. Then proceed with your development plan

This step is **mandatory** before writing runtime, storage, restore, or other non-UI code.
